# BUG-003: require_user() Navigation Mismatch + Connection Leak

## Summary
`require_user()` in `auth.py` called `st.switch_page("pages/0_Login.py")` — same NAV-001 pattern.
Also leaked the DB connection passed to it (never closed before st.stop()).

## Trigger Scenario
User is authenticated, session is deleted from DB (e.g., they log in on another device — single-session
enforcement deletes their old session). Next action on original device: `require_user()` finds no
valid session, tries to redirect to Login, but Login isn't in the authenticated nav. Crash.

## Two Bugs in One
1. NAV-001: `st.switch_page("pages/0_Login.py")` when Login not in current navigation
2. Connection leak: `conn` was passed to `require_user()` from the calling page. After `st.stop()`,
   execution halts — the calling page's `conn.close()` at the bottom is never reached.
   Affected 8 pages: Dashboard, 2_Quiz, 3_Study_Brief, 4_My_Progress, 5_Return_Quiz,
   8_My_Setup, 9_Answer_Review, 10_Profile.

## Fix Applied
`web/auth.py` lines 130-136:
- Added `conn.close()` before navigation
- Pop `session_token` and `user_id` from session_state (clears the invalid session so `_authed=False`)
- Replace `st.switch_page()` + `st.stop()` with `st.rerun()`

## Files Changed
- `web/auth.py` lines 124-136
