# BUG-023 — Profile "Answers Graded" undercounts — misses UPSC descriptive attempts

**Status:** FIXED  
**Severity:** MEDIUM  
**Category:** data-sync / cross-db  
**Session:** S28  
**Commit:** TBD

## Symptom
`/profile` "ANSWERS GRADED" stat only reflects IES descriptive attempts. If a user has submitted UPSC descriptive answers, those are not counted.

## Root Cause
`profile_bp.py` queries `descriptive_attempts` via `get_conn()` (ies.db only). UPSC has its own `descriptive_attempts` table in upsc.db. Same SYNC-001 pattern as BUG-021 and BUG-022: profile route has no blueprint-scoped connection to upsc.db.

## Fix
Same approach as BUG-021 MCQ fix: open a short-lived direct connection to upsc.db in the profile route, query its `descriptive_attempts`, and sum:

```python
answers_graded = ies_answers + upsc_answers
```
