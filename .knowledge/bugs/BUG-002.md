# BUG-002: Logout Crashes — Login Page Not in Authenticated Navigation

## Summary
Clicking "Sign out" on Profile page throws "Page not found: pages/0_Login.py". Users cannot log out.

## Root Cause
When user is authenticated, `app.py` registers the full nav set which does NOT include `0_Login.py`.
`Profile.py` calls `st.session_state.clear()` then `st.switch_page("pages/0_Login.py")`. By this
point, Login is not in the current navigation registration. Identical pattern to BUG-001.

## Fix Applied
`web/pages/10_Profile.py` line 189: replaced `st.switch_page("pages/0_Login.py")` with `st.rerun()`.
Also moved `conn.close()` to run BEFORE `st.session_state.clear()` (so the conn reference is valid).
After clear, `_authed=False` on rerun → app.py registers `[_login]` → routes to login.

## Files Changed
- `web/pages/10_Profile.py` lines 184-189

## Note
`conn.close()` must happen before `session_state.clear()` in logout flows — the conn object lives
in a local variable, not session_state, so this is safe. But clearing first and then trying to close
is also safe since it's a local variable reference, not a session_state reference.
