# BUG-008: OAuth CSRF — State Parameter Generated But Never Validated

## Summary
`build_auth_url()` in `auth.py` generates a `state=secrets.token_hex(16)` parameter. This is the
CSRF protection for OAuth. However, `0_Login.py` never stores the generated state and never validates
the `state` in the callback response against what was sent.

## Risk
A malicious site could craft a redirect to the OAuth callback with a forged `?code=...`. Without
state validation, the app would process it. Low risk at current scale — Google's OAuth codes are
one-time-use and tied to the redirect_uri.

## Recommended Fix
1. At `build_auth_url()` time: store state in `st.session_state._oauth_state = state`
2. In `0_Login.py` callback: validate `params.get("state") == st.session_state.get("_oauth_state")`
3. On mismatch: show error, clear params, do not process code

## Priority
Low. Implement before public launch with non-invited users.
