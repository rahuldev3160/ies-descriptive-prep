# BUG-001: OAuth Callback Crashes — Dashboard Not in Navigation

## Summary
Production app showed "Page not found: pages/Dashboard.py" immediately after Google OAuth sign-in.

## Root Cause
`app.py` evaluates `_authed = bool(st.session_state.get("session_token"))` at boot and registers
`st.navigation([_login])` (login-only) when unauthenticated. The OAuth callback in `0_Login.py`
then sets `session_token` and calls `st.switch_page("pages/Dashboard.py")`. Streamlit throws because
Dashboard is not in the currently registered navigation set.

## Analogy
A hotel receptionist prints the room access list once at check-in. The guest is told "you only have
lobby access." A staff member then gives the guest a room key and tells them to go to Room 101 —
but the receptionist's list never updated, so the door system rejects the request.

## Fix Applied
Replaced both `st.switch_page("pages/Dashboard.py")` calls in `web/pages/0_Login.py` (lines 24, 47)
with `st.rerun()`. The rerun re-executes `app.py` which now sees `_authed=True` and registers
Dashboard as the default page.

## Files Changed
- `web/pages/0_Login.py` lines 24, 47

## How to Detect in Future
Any `st.switch_page("pages/X.py")` call where X is only in the AUTHENTICATED navigation branch,
but the call happens BEFORE auth is confirmed in the same execution cycle.
