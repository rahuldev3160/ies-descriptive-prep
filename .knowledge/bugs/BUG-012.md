# BUG-012: 1_Model_Answers.py + 7_UPSC_Mains.py Have No require_user() — Intentional?

## Summary
These two pages do not call `require_user()`. They are accessible to unauthenticated users via direct
URL. The content they serve (model answers, UPSC study material) is reference/shared content with no
user-specific data writes.

## Assessment
Likely intentional — these are read-only reference pages. Not a security risk since the data is not
sensitive and no user writes occur.

## Action Needed
Confirm intent with Rahul. If intentional: add a comment in each page header noting this is public.
If NOT intentional: add `require_user(conn)` at the top of each and handle the conn.close() properly.
