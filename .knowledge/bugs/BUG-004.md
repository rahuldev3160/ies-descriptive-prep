# BUG-004: Session State Bleed Between Users on Same Browser

## Summary
When User B logs in on the same browser tab/session as User A, User A's quiz evaluations, rate-limit
counters, RBI drill progress, and quiz results remain in `st.session_state`. User B sees stale data.

## Analogy
An exam hall where the whiteboard isn't erased between candidates. The new student sits down and sees
the previous student's working still on the board.

## Affected State Keys (not cleared on login)
- `quiz_*` (quiz_session_evals, quiz_last_eval, quiz_count, quiz_window_start, quiz_curr_qid, quiz_batch_result)
  - Risk: User B inherits User A's AI evaluation feedback AND rate limit window (could block B immediately)
- `rq_*` (rq_paper_id, rq_topic_id, rq_last_result, rq_session_id)
  - Risk: User B's Return Quiz loads with A's selected paper/topic and sees A's last result
- `rbi6_*` + `rbi_drill_*` (8+ keys)
  - Risk: User B sees A's completed RBI buckets and drill scores

## Fix Applied
`web/pages/0_Login.py` after successful OAuth: added cleanup before setting new user's session_token.
```python
_stale = [k for k in st.session_state if k.startswith(("quiz_", "rq_", "rbi6_", "rbi_drill_"))]
for k in _stale: del st.session_state[k]
```

## Files Changed
- `web/pages/0_Login.py` lines ~40-42 (before session_token assignment)
